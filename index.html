NAND结构及特点

发表于 2024-05-30 分类于 存储开发 ， Flash

1、存储结构

NAND Flash 从物理结构上进行划分，可以从芯片层拆分到晶体管：

  • CHIP：一个闪存芯片就是一个 CHIP，也称为 DEVICE，一个芯片中包含了若干个 DIE。
  • DIE：DIE 是指芯片晶圆上的小方块，每个 DIE 包含了若干个 Plane。
  • Plane：Plane 是一个存储矩阵，包含若干个 Block。
  • Block：Block 是 NAND 中擦除操作的最小单位，包含若干个 Page。
  • Page：Page 是NAND 中读、写操作的最小单位，每个 Page 中包含了成千上万个 Cell。
  • Cell：Cell 主要是浮栅 MOS 管，用来存储信息。

2、排列结构

存储列阵：NAND 中的存储单元按照矩阵式排布，将每一行中所有存储单元的控制栅极串联起来形成一条链，这条链被连接到字线（Word Line）上；将每一列中存储单元的源极向下连接到相邻存储单元的漏极，第一个存储单元的漏极连接到位线（Bit Line）上，最后一个存储单元的源极连接到源线（Source Line）上。

eMMC基础知识

发表于 2024-05-30 分类于 存储开发 ， eMMC

1、eMMC基本组成

eMMC（Embedded Multi Media Card）是一种内嵌式存储器标准规格，主要针对手机、平板电脑等移动设备设计。它主要由三部分构成：

1.1 NAND Flash

NAND Flash 作为一种非易失性存储器，在嵌入式系统中通常用于存放系统、应用和数据。eMMC 在内部对 NAND Flash 划分为以下四个主要区域：

  • BOOT Area Partition 1 & 2：这两个分区主要是为了支持eMMC 启动系统而设计。
  • RPMB Partition (Replay Protected Memory Block)：这个分区通过一些处理保证其内部的数据不被非法篡改，通常用于保存安全相关的数据，如指纹数据，安全支付相关的密钥。
  • General Purpose Partition 1 ~ 4 (GPP)：这个分区主要用于存储系统或者用户数据，一般需要在 eMMC 出厂后由用户主动配置后才会存在。
  • User Data Area (UDA)：这个分区主要用于存放系统和用户数据，通常 UDA 占据了 eMMC 总容量的 93% 左右，是用于读写的主要区域。

1.2 Device Controller

在单独使用 NAND Flash 接入到主机时，需要主机端有 NAND Flash Translation Layer，即 NFTL 或 NAND Flash 文件系统来做坏块管理、磨损均衡等功能。

Code

1.排序

种花问题

发表于 2024-05-16 更新于 2024-05-17 分类于 数据结构与算法 ， leetcode

原题

  • 种花问题

Code

1.贪心

方法与思路：假设花坛中有一段连续的地块没有花，连续地块的数量为 n ，这个连续的地块最多可以种植 ( ( ( n - 2 ) + 1 ) / 2 ) 朵花；但如果地块处于花坛的头部或者尾部，则最多可以种植 ( ( ( n - 1 ) + 1 ) / 2 ) 朵花，那么只需要找出花坛中有多少个这样的连续地块并获取每一个连续地块的数量 n 进行计算即可；

  • 为了统一计算方式为 ( ( ( n - 2 ) + 1 ) / 2 )，我们可以在找到头部或者尾部地块时多加一个地块，在计算时扣除掉即可

复杂度分析：

  • 时间复杂度：O(n)
  • 空间复杂度：O(1)

两个列表的最小索引总和

发表于 2024-05-15 更新于 2024-05-17 分类于 数据结构与算法 ， leetcode

原题

  • 两个列表的最小索引总和

Code

1.哈希+条件查找

方法与思路：构建一个哈希表，将餐厅名作为键，索引作为值，将第一个数组映射为哈希表；对第二个数组进行遍历，在遍历每一个元素时，都对哈希表进行查找，哈希表中是否存在与当前元素相同的键，如果存在则获取两个元素的索引和，并判断该索引和是否为满足条件索引和的最小值，如果是最小值，则更新最小值并重置数组，如果与之前的最小值相同，则直接将该元素插入。

复杂度分析：

  • 时间复杂度：O(n)，n 为两个数组中较长数组的长度
  • 空间复杂度：O(n)，n 为两个数组中较短数组的长度

Code

1.原理计算

  • 空间复杂度：O(1)

最长和谐子序列

发表于 2024-05-15 更新于 2024-05-24 分类于 数据结构与算法 ， leetcode

原题

  • 最长和谐子序列

Code

1.哈希+条件查找

方法与思路：首先构建一个哈希表，键为原数组元素的值，值为该元素出现的次数；将原数组映射为哈希表后，对哈希表进行遍历，在遍历每一个元素时，查询哈希表中是否存在该元素的伙伴，如果存在则获取两个元素出现次数的和，满足条件和中的最大值即为最长子序列的长度。

  • 如果 b = a + 1，则称 b 为 a 的伙伴

复杂度分析：

  • 时间复杂度：O(n)，n 为 nums 的长度
  • 空间复杂度：O(n)，n 为 nums 的长度

分糖果

发表于 2024-05-15 更新于 2024-05-24 分类于 数据结构与算法 ， leetcode

原题

  • 分糖果

Code

1.哈希+计数

方法与思路：需要在尽可能少的糖果中吃到尽可能多的类型，我们可以遍历数组，每遇到一种不同的糖果就吃一颗，并且对吃的颗数进行计数；那么出现了子问题：怎么判断糖果的类型不同？可以使用哈希表进行去重，当哈希表中已经记录了这颗糖果时就代表已经吃过了，没记录就代表没吃过；最后在计数的同时判断有没有超过医生建议的数量即可。

复杂度分析：

  • 时间复杂度：O(n)，n 为数组长度
  • 空间复杂度：O(n)，n 为构建的哈希表大小

2.排序+计数

方法与思路：先将数组进行排序，数组中的糖总是大于两颗的，所以第一颗一定是可以吃的，那么从第二颗开始判断，如果与上一颗糖不同则可以吃，当吃过的糖果数等于医生建议时或数组被遍历完后，返回吃过的不同糖果数。

原题

  • 重塑矩阵

Code

1.flatten映射

方法与思路：用两个下标进行标注，遍历原数组的元素插入重塑数组，当重塑数组的第一行插入满了以后切换进入下一行，直到插入完毕为止，先将原数组映射为一维数组，获得其下标，将下标再映射为目标数组，因为映射计算是可逆的，所以可以直接将原数组赋值给目标数组。
