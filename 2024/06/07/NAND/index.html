NAND基本知识

发表于 2024-06-07 分类于 存储开发 ， NAND

1、NAND概述

1.1 基本概念

NAND 又称 NAND 闪存，是闪存（一种非易失性存储技术，该技术可以在没有电流供应的情况下，长时间保存数据，被广泛用于各种电子设备中）的一种类型，因其具有较高的存储密度和较低的成本，在大规模存储中被广泛应用，如 SSD，存储卡等。

1.2 存储原理

NAND 基于 Flash Memory Cell 技术实现，这种技术使用了浮栅晶体管（Floating Gate Transistor）来存储数据，一个晶体管被称为一个 Cell（基本存储单元），其结构如图所示：

  • Control gate（控制栅极）
  • Isolator（绝缘层）
  • Floating gate（浮栅）
  • Source（源极）
  • Drain（漏极）
  • n-channel（n沟道）
  • Substrate（衬底）

浮栅中可以存储电荷，其上下两层绝缘层可以保证晶体管断电后浮栅内部的电荷几乎不流失，通过将电荷存储进浮栅中来存储二进制信息，这就是浮栅晶体管的存储原理。闪存则将多个浮栅晶体管组合排列，将复杂的信息转换为二进制信息后，存储到每个晶体管中，这就是 NAND 的工作原理。

1.3 读擦写原理

读原理：通过在控制栅极施加读取电压，衬底接地，将浮栅中的电荷吸引到沟道中，但不足以发生隧穿效应，此时测量漏极和源极中是否存在大电流，即晶体管是否导通；若未导通，说明阈值电压较高，该状态读取为 0；若导通，说明阈值电压较低，该状态读取为 1。

擦除原理：通过在控制栅极施加低电压，衬底加高电压，使浮栅中的电荷发生隧穿效应到达源极或衬底中，浮栅中没有了存储的电荷，也就没有了存储的信息，此时控制栅极的阈值电压较低，施加读取电压足以让晶体管导通，这个状态通常被读取为二进制 1，也是晶体管的默认状态。

写原理：通过在控制栅极施加高电压，衬底接地，使衬底中的电荷到达沟道中，并发生隧穿效应到达浮栅中，此时控制栅极的阈值电压较高，施加读取电压不足以使晶体管导通，这个状态通常被读取为二进制 0。

2、组织结构

2.1 阵列组织

NAND 中的存储单元排列是一种串联方式，其串联结构如图所示：

  • WL (Word Line) ：每一行的浮栅晶体管的控制栅极通过串联的方式连接到字线上。
  • BL (Bit Line) ：每一列的第一个晶体管的漏极连接到位线上。
  • SL (Source Line) ：所有列的最后一个晶体管的源极连接到源线上。

其中每一行就是一个 Page，整个存储阵列就是一个 Block，它的好处在于不需要给每一个晶体管接入字线、位线和源线，去除了大量的金属导线，使得相同面积下可以集成更多的存储单元，提高了存储密度，且一定程度上简化了生产工艺，降低了生产成本。

2.2 物理结构

NAND 的物理结构示意图如下：

  • CHIP：通常指一个闪存芯片，也称为 DEVICE，一个 CHIP 中包含若干个 LUN（也称为 DIE）。
  • LUN：LUN 是芯片晶圆上的小方块，一个 LUN 中包含若干个 Plane。
  • Plane：Plane 是 LUN 中的一个存储单元阵列，一个 Plane 中包含若干个 Block。
  • Block：Block 是 Plane 的基本单位，同时也是擦除操作的最小单位，一个 Block 中包含若干个 Page。
  • Page：Page 是 Block的基本单位，同时也是读、写操作的最小单位，一个 Page 中包含了成千上万个 Cell。

NAND Flash 的物理结构大致划分为以上部分，根据不同厂商和制造商的要求，可能还会包含一些其他的结构，如 Plane 可能还包含 cache register 和 page register 以优化读写性能。

2.3 寻址方式

NAND 中为每个 LUN 、Plane、Block 和 Page 都进行了编号，这些编号就是它们的地址，将它们的地址按照高位到地位组织成一个数据，这个数据在逻辑上被称为行地址（Row Address），其结构如图所示：

而在 Page 中又由众多的 Cell 组成，每个 Cell 都有一个独特的地址，这个地址在逻辑上被称为列地址（Column Address）。通过行地址和列地址就可以将数据存储到指定位置或从指定位置读取数据。

3、存储模式

NAND 根据存储单元可以存储信息的位数可以分为以下模式：

存储模式 比特数/单元 擦写寿命 读取速度 写入速度 容量密度
SLC (Single-Level Cell) 1 高 快 快 低
MLC (Multi-Level Cell) 2 中 较快 较快 较高
TLC (Triple-Level Cell) 3 较低 较慢 较慢 高
QLC (Quad-Level Cell) 4 低 慢 慢 很高

4、接口模式

NAND 的接口主要可以分为两种，ONFI (Open NAND Flash Interface) 和 Toggle-Mode。Toggle-Mode 标准由三星和东芝开发。ONFI 标准则是由Intel、Micron、Hynix、Phison等厂商共同开发。以下是它们的主要区别：

特性 ONFI Toggle-Mode
传输模式 DDR（ONFI 2.x起） DDR
同步信号 CLK DQS
信号线路 单端信号 单端信号
协议复杂性 相对复杂 相对简单
接口兼容性 高，广泛适用 相对较低，但适用特定高性能场景
性能 支持较高的数据速率 高速数据传输，特别适合高性能和移动设备应用

5、命令操作

NAND 闪存操作通常需要通过发送不同的命令来执行，这些命令包括读取、写入、擦除、复位等操作。以下是 NAND 闪存中常见的一些命令及其功能：

  1. 读取命令（Read）：从指定地址读取数据。
  2. 编程命令（Program）：将数据编程到指定地址。
  3. 擦除命令（Erase）：擦除指定块中的数据。
  4. 复位命令（Reset）：复位 NAND 闪存芯片。
  5. ID 查询命令（Read ID）：读取 NAND 闪存的制造商和设备 ID。
  6. 写保护命令（Write Protect）：设置或解除写保护状态。
  7. 锁存转储命令（Latch Dump）：将页面寄存器中的数据转储到数据寄存器。
  8. 随机缓存读取命令（Random Cache Read）：预加载下一页数据以提高读取速度。
  9. 快速读取命令（Fast Read）：以更快的速度读取数据。
  10. 自检命令（Read Status）：读取 NAND 闪存的状态寄存器，用于检测操作是否完成或错误状态。

5.1 基本时序

读操作时序：

  • 发送读取命令：告知设备将要读取数据。
  • 发送地址信息：告知设备读取数据的位置。
  • 发送确认命令：告知设备可以开始准备数据。
  • 检查状态寄存器：当设备开始将传输数据时，设备会进入忙碌状态，当设备状态切换到空闲状态时，可以开始读取数据。
  • 读取数据：发送 RE 信号并到 DQ 总线上读取数据。

写操作时序：

  • 发送写入命令：告知设备将要写入数据。
  • 发送地址信息：告知设备数据写入的位置。
  • 发送数据：发送 WE 信号并将要写入的数据发送到 DQ 总线上。
  • 检查状态寄存器：当设备将数据传出到存储阵列时，设备会进入到忙碌状态，当设备切换到空闲状态时，说明数据写入完成。

擦除操作时序：

  • 发送擦除命令：告知设备将要擦除数据
  • 发送地址信息：告知设备将要擦除数据的位置。
  • 检查状态寄存器：当设备开始擦除时，设备会进入到忙碌状态，当设备切换到空闲状态时，说明擦除完成。

复位操作时序：

  • 发送复位命令：告知设备进行复位操作，并将设备状态置为空闲。
  • 检查设备状态：当设备进入空闲状态说明复位成功，

以上命令操作都是最简单的操作，在 K9AFGD8J0C 中定义了命令集并实现了复杂的操作如：随机缓存读取操作、顺序缓存读取操作、缓存写入操作、回写操作等等，数据手册中说明了详细的操作步骤以及时序要求。

5.2 内部行为

NAND 在接收到主机发送的数据后，会根据数据的类型做出一系列的动作，主要有：

  • 命令锁存：将接收的命令存储到命令寄存器中。
  • 命令识别：对命令进行识别，并根据不同的命令触发不同的行为。
  • 地址锁存：将接收的地址信息存储到内部缓存中。
  • 内部寻址：根据接收到的地址信息进行寻址。
  • 数据锁存：将接收到的数据锁存到数据寄存器中。
  • 加载数据：将存储阵列中的数据加载到页寄存器中。
  • 缓存数据：将页寄存器中的数据加载到缓存寄存器中。
  • 预加载：将页寄存器的数据加载到缓存寄存器中后，将下一次要读取的数据提前加载到页寄存器中。
  • 切换状态：根据内部行为，将 R/B 置为忙碌或空闲。
  • 复位：清除命令寄存器和地址寄存器中的数据，将设备状态置为空闲，初始化计数器和内部逻辑等等。

这些内部行为都由 NAND 内部的硬件电路控制，作为主机不需要关心其内部实现，控制好操作命令及时序即可对其进行读写。
