eMMC分区管理

发表于 2024-06-01 更新于 2024-06-06 分类于 存储开发 ， eMMC

1、分区概述

eMMC（嵌入式多媒体卡）内部将 NAND 划分为四个主要区域，最多可以分为 8 个分区。下图展示了 eMMC 内部的分区结构：

  • 引导分区（Boot Area Partitions, BAP）
  • RPMB分区（Replay Protected Memory Block, RPMB）
  • 通用分区（General Purpose Partitions, GPP）
  • 用户数据区（User Data Area, UDA）

2、BAP

BAP 分区主要用于存储启动代码和关键数据，以支持 SOC 启动系统。通常 eMMC 会包含两个分区（BAP1 & BAP2），以提升设备的冗余性和可靠性。BAP 分区的配置和管理通过 EXT_CSD Register 寄存器中的相关字段来完成：

  • BOOT_SIZE_MULT：定义每个启动分区的大小，单位为 128KB。
  • BOOT_CONFIG：配置启动模式和启动分区的选择。

除了这些字段以外，还有一些其他的字段会直接或间接的影响到 BAP 分区的使用，具体可以参考 EXT_CSD Register 相关文档。

如果需要配置 BAP 分区，具体配置步骤如下：

  1. 读取当前配置：使用适当命令读取 EXT_CSD Register 的当前值。
  2. 设置 BOOT_SIZE_MULT：根据需求配置 BAP 分区的大小。
  3. 配置 BOOT_CONFIG：配置启动模式和启动分区。

一般出厂后 BAP 分区的大小是固定的，通常为 4M，但有的芯片厂家会提供功能，让用户自行配置 BAP 分区的大小。

3、RPMB

RPMB 是 eMMC 存储器中的一个特殊分区，设计用于存储敏感数据，并提供防重放攻击和数据篡改保护。RPMB 分区的配置和管理也是通过 EXT_CSD Register 中的相关字段来完成：

  • RPMB_SIZE_MULT：指定 RPMB 分区的大小，单位为 128KB。
  • RPMB_WRITE_COUNTER：用于跟踪 RPMB 分区的写操作计数器，每次写操作都会增加计数器值，防止重放攻击。这个字段不是直接通过 EXT_CSD Register 访问，而是在执行 RPMB 相关命令时，通过响应包获取。

除了这些字段以外，还有一些其他的字段会直接或间接的影响到 RPMB 分区的使用，具体可以参考 EXT_CSD Register 相关文档。

在使用的 RPMB分区之前，需要进行配置和初始化，以确保其安全特性和功能能够正常工作。这包括设置认证密钥和初始化写计数器等步骤：

  1. 设置认证密钥：生成一个认证密钥（认证密钥是用于生成和验证 HMAC 的密钥，通常由设备的安全模块生成），使用特定的 eMMC 命令将认证密钥写入 RPMB 分区。
  2. 初始化写计数器：读取当前的写计数器值，确保其初始化为 0 或其他预定义值，并通过特定命令验证写计数器值。
  3. 数据写入和认证：计算待写入数据的 HMAC，将数据和 HMAC 一起写入 RPMB 分区，读取并验证 HMAC，以确认写操作的完整性。

在使用 eMMC 的 RPMB 分区之前，进行适当的配置和初始化是确保其安全特性和功能正常工作的关键步骤。配置认证密钥、初始化写计数器以及确保数据写入和认证的过程，是保护敏感数据和防止重放攻击的必要措施。通过这些步骤，可以确保 RPMB 分区在数据存储和安全性方面的高效性和可靠性。

4、GPP

通用分区（General Purpose Partition，GPP）是 eMMC 中的一种分区类型，用于为特定的应用和功能提供灵活的存储空间，用以数据隔离，提高数据的安全性和稳定性。GPP 分区的配置和管理也是通过 EXT_CSD Register 中的相关字段来完成：

  • GP_SIZE_MULT_x：这里的 x 是指 GPP 的分区（1-4），这是一个 24 位值，占用三个字节，它指定 GPP_x 分区的大小，单位为 512KB。

除了这些字段以外，还有一些其他的字段会直接或间接的影响到 GPP 分区的使用，具体可以参考 EXT_CSD Register 相关文档。

5、UDA

用户数据区是 eMMC 中用于存储所有用户数据的主要区域。它负责存储操作系统、应用程序和用户文件，并通过各种技术手段来优化性能和保障数据安全。合理配置和管理 UDA 可以提高设备的存储效率和数据可靠性。它没有一些独立的控制字段，但是一些其他的字段也会间接的影响到 UDA 分区的使用，具体可以参考 EXT_CSD Register 相关文档。
