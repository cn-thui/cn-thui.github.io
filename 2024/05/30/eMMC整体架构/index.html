eMMC整体架构

发表于 2024-05-30 更新于 2024-06-05 分类于 存储开发 ， eMMC

1、架构概述

eMMC（嵌入式多媒体卡）整体结构可以分为三个部分。下图展示了 eMMC 整体架构：

  • Standard Interface：用于与外部主机通信的标准接口
  • Device Controller：用于管理通信和存储的控制单元
  • NAND Flash：用于存储数据的存储单元

eMMC 由三部分构成，使得它具有高集成度、简化设计、易于集成和维护等多方面的优势，这使得eMMC成为嵌入式系统和移动设备中广泛应用的存储解决方案。

2、标准接口

eMMC 基于 MMC 标准，继承了 MMC 的接口规范，并在其基础上进行优化以适应嵌入式系统的要求，下图展示了 eMMC 的接口标准中规定的总线：

  • CLK：用于接收 Host 输出的时钟信号。
  • Reset：用于接收 Host 输出的复位信号。
  • CMD：用于 Host 向 eMMC 发送命令信号和 eMMC 向 Host 发送应答信号。
  • Data：用于 Host 和 eMMC 间的数据通信，同时用户可以配置使用的 Date 线的数量。
  • DS (Data Strobe)：用于 HS400 模式下数据接收的同步，提高数据传输的稳定性，在 eMMC 5.0 规范引入。

eMMC 接口相对于 MMC 来说，在数据传输、兼容性等方面进行优化，确保主机和 eMMC 设备之间能够进行高效的通信。

3、控制单元

控制单元是整个 eMMC 的核心，它向上负责给主机提高标准化的接口，向下负责管理 NAND 的各种操作。下图展示了控制单元实现的部分功能：

  • 数据传输：提供数据缓冲区，用于临时存储正在传输的数据，以提高整体读写性能。
  • 错误检测：在其内部使用 ECC (Error Correction Code) 机制检测错误并纠正。
  • 地址转换：将主机提供的逻辑地址转换为 NAND 的物理地址。
  • 磨损均衡：磨损均衡算法确保所有存储单元均匀使用，防止某些单元过早磨损，以延长 NAND 的寿命。
  • 坏块管理：自动检测和管理坏块，确保数据的完整性和可靠性。
  • 安全功能：提供访问控制机制，防止未经授权的读写操作。
  • 电源管理：支持多种电源模式（如休眠模式、空闲模式）以降低功耗。
  • ………

eMMC 的控制单元通过实现以上功能，为外部主机提供高效、可靠、安全的数据存储和访问。

4、存储介质

eMMC 内部将其存储介质 NAND 划分为四个主要区域，最多可分为 8 个分区。下图展示了 eMMC NAND 内部的具体分区：

  • BOOT Area Partition 1 & 2：存储启动加载程序和引导相关数据，确保设备能够正确启动操作系统。
  • RPMB Partition (Replay Protected Memory Block)：提供安全存储区域，用于防止重放攻击，保护关键数据的完整性和安全性。
  • General Purpose Partition 1 ~ 4：用于特定应用或数据存储的自定义分区，灵活配置以满足不同应用需求。
  • User Data Area：存储用户数据、应用程序和操作系统文件，是eMMC中最大的分区，适应各种存储需求。

eMMC 通过这种分区方式，确保 eMMC 能够高效、安全地存储和管理各种类型的数据。
