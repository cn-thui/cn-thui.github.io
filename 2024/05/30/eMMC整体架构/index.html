eMMC 整体架构

eMMC（Embedded Multi Media Card）是一种内嵌式存储器标准规格，主要针对手机、平板电脑等移动设备设计。它主要由三部分构成：

  • Standard Interface：用于通信的标准接口
  • Device Controller：用于管理的控制单元
  • NAND Flash：用于存储数据的存储单元

这三个组件共同工作，构成了 eMMC 存储解决方案的关键部分，确保设备能够高效、可靠地进行数据存储和交换。

1 NAND Flash

eMMC 在内部对 NAND Flash 划分为以下四个主要区域：

  • BOOT Area Partition 1 & 2：存储启动加载程序和引导相关数据，确保设备能够正确启动操作系统。
  • RPMB Partition (Replay Protected Memory Block)：提供安全存储区域，用于防止重放攻击，保护关键数据的完整性和安全性。
  • General Purpose Partition 1 ~ 4：用于特定应用或数据存储的自定义分区，灵活配置以满足不同应用需求。
  • User Data Area：存储用户数据、应用程序和操作系统文件，是eMMC中最大的分区，适应各种存储需求。

这些分区共同确保eMMC能够高效、安全地存储和管理各种类型的数据。

2 Device Controller

在单独使用 NAND Flash 接入到主机时，需要主机端有 NFTL (NAND Flash Translation Layer) 对 NAND Flash 进行管理：

而 eMMC 在其内部集成了 Device Controller 替代 NFTL，并通过一系列复杂的算法和机制，确保eMMC能够高效、可靠地工作，同时提供各种安全和性能优化功能。相比于直接将 NAND Flash 接入到主机，eMMC 屏蔽了 NAND Flash 的物理特性，节省了主机 CPU 资源的同时减少了主机系统软件的复杂度。

3 Standard Inetrface

eMMC 基于 MMC 总线，继承了 MMC 的接口规范，并在其基础上进行优化以适应嵌入式系统的要求，eMMC 的接口标准规定的信号线有：

  • CLK：用于接收主机输出的时钟信号。
  • Reset：用于接收主机输出的复位信号。
  • CMD：用于主机向 eMMC 发送命令信号和 eMMC 向主机发送应答信号。
  • Data：用于主机和 eMMC 间的数据通信，同时用户可以配置使用的 Date 线的数量，可以选择 4 根或者 8 根。
  • DS (Data Strobe)：用于 HS400 模式下数据接收的同步，提高数据传输的稳定性，在 eMMC 5.0 引入。

eMMC 的标准接口在在数据传输、兼容性等方面进行了优化，确保主机和 eMMC 设备之间能够进行高效的通信。
